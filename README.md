## Quick start

```bash
git clone <repository-url> my-workspace/ent
cd my-workspace
node ent/tools/ent.mjs scaffold --workspace-root .
```

`scaffold` runs `npm install` in `ent/` for the bundled WordPress MCP bridge (no runtime `npx`).

In Cursor, run **`/ent-onboard`**. Open the **parent folder** that contains `ent/`, not the kit repo alone.

## Session preload

| Layer | What | When it runs |
|-------|------|----------------|
| **Always-on rules** | Thin project brief — what Ent is, where changes belong, extension map, anti-patterns | Every chat (`.cursor/rules/00-ent-preload.mdc`) |
| **Session hook** | Fresh snapshot — open mode, audit/onboard status, **where to start this session** | Once per new Composer chat (`sessionStart` → `session-boot.mjs`) |
| **Fallback hooks** | Ensure `.ent/session-context.md` exists if the session hook misfires | First user message / first tool use |

**Cursor alone** loads system prompt, tools, MCP, skills, and any rules you added — but not live project status or a consistent “start here” map unless you maintain it. **With Ent**, `ent sync` projects adapters from `agent-adapters/` into workspace `.cursor/`; the hook reads audit state and writes `.ent/session-context.md` on the consumer root.

Claude Code gets the same intent via synced project fragments and `.mcp.json` — different packaging, same preload goal.

**Source of truth:** edit `agent-adapters/cursor/workspace-template/`, then sync. Do not edit generated workspace `.cursor/` in place.

**Caveat:** preload amplifies structure; it does not organize a messy repo. Decide what lives where before freezing it into rules.

## CLI

```bash
node ent/tools/ent.mjs validate-manifest
node ent/tools/ent.mjs sync --workspace-root . --agent cursor
node ent/tools/ent.mjs onboard --workspace-root . [--log]
node ent/tools/ent.mjs offboard --workspace-root . [--dry-run]
node ent/tools/ent.mjs audit --workspace-root .
node ent/tools/ent.mjs render-onboard --workspace-root .
node ent/tools/ent.mjs scaffold --workspace-root .
```

After adapter changes, re-sync so preload rules and hooks reach the workspace.

## Layout

| Path | Role |
|------|------|
| `ent/` | Immutable kit — update via `git pull` |
| `content/` | Mutable site content |
| `.ent/` | Audit state, onboard checklist, `session-context.md` |
| `.env` | Workspace credentials |
| `.cursor/` | Generated by `ent sync` — rules, hooks, MCP |
